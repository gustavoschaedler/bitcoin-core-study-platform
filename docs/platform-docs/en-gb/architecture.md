# Platform Architecture

## Overview

The Signet Core Study Platform runs as a set of Docker containers orchestrated by Docker Compose. All services communicate over an internal Docker network.

## Services

### bitcoind

The Bitcoin Core node running on **Signet** (the official test network with a signature challenge). Exposes JSON-RPC on port `38332` and ZMQ on ports `28332`–`28335`.

### web

FastAPI application serving the web UI and API endpoints. Routes include:

- `/` — Home dashboard with node status
- `/faucet` — Request test coins
- `/mempool` — Live mempool explorer
- `/wallet` — Wallet lab (create, manage and export wallets)
- `/signing` — Signing lab (create PSBTs, sign and broadcast transactions)
- `/stats` — Container resource monitoring
- `/study-docs` — This documentation viewer

### redis

Caching layer for RPC responses, rate limiting, and faucet cooldown tracking.

### display

HDMI dashboard optimised for a connected screen, showing real-time node metrics.

### terminal-webui / terminal-proxy

Browser-based terminal with direct `bitcoin-cli` and JSON-RPC access to the Signet node.

## Authentication Modes

The platform supports two RPC authentication methods configured via `BITCOIN_RPC_AUTH_MODE`:

- **password** — traditional `rpcuser` / `rpcpassword` credentials
- **cookie** — reads the `.cookie` file generated by Bitcoin Core at startup

## Data Flow

```text
Browser ──► FastAPI (web) ──► Bitcoin Core RPC (bitcoind)
                           ──► Redis (cache)
                           ──► Docker API (stats)
```

All RPC calls go through `apps/web/app/core/rpc.py`, which handles both authentication modes and connection pooling via `httpx`.
